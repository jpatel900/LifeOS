# Role-Separated Delivery Checkpoints

- Task name and branch: Add role-separated delivery checkpoints to agent workflow; `main`
- Original scope: Governance/process docs and prompt/template updates only for Issue #42. No workflow YAML edits, runtime behavior changes, new dependencies, or orchestration additions.
- Assumptions:
  - Role separation must stay an engineering-process concept, not a LifeOS runtime capability.
  - The existing GitHub-first issue -> PR -> CI/review path remains the canonical delivery flow.
  - PR review prompts cannot safely rely on PR-authored restatements of issue scope.
- Decisions:
  - Added `docs/agent/ROLE_SEPARATED_DELIVERY.md` as the canonical checkpoint guide for research, spec, build, test, and validation.
  - Added a required `Research/spec checkpoint` field to `.github/ISSUE_TEMPLATE/agent-task.yml`.
  - Updated `docs/agent/CODEX_PROMPT_TEMPLATE.md` so medium/high-risk or cross-flow prompts explicitly carry checkpoint context plus the Verification Oracle.
  - Updated both PR review prompts to compare diffs against issue acceptance criteria, research/spec checkpoint, and forbidden changes when trusted issue context exists, and to report the gap when it does not.
- Deviations:
  - Did not change `.github/workflows/codex-pr-review.yml` because Issue #42 forbids workflow changes and the prompt-only change was sufficient.
- Tradeoffs:
  - Without workflow-side issue-context injection, reviewers cannot always perform a fully trusted issue-to-diff comparison. The prompt now makes that limitation explicit instead of faking certainty.
  - Keeping this as docs/prompt/template guidance avoids accidental weakening of existing automation gates.
